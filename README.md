# Discrod_Role_Bot

Kuruma собрал этот бот под один четкий сценарий: после пересоздания ролей на основном Discord-сервере быстро и безопасно выдать выбранную роль всем участникам без тяжелой миграционной панели и без ручного кликанья по 500 людям.

## Что делает бот

- Дает 2 slash-команды: `/status` и `/grantrole`.
- `/grantrole` открывает встроенный Discord role picker, считает preview и просит подтверждение кнопкой.
- После подтверждения бот проходит участников по безопасной очереди, не пытается спамить API и пишет JSON-отчет.
- Повторный запуск безопасен: участники, у которых роль уже есть, пропускаются.

## Команды

### `/status`

Показывает:

- активную задачу, если идет выдача роли;
- скорость обработки;
- примерный ETA;
- итог последней завершенной задачи;
- путь к последнему JSON-отчету.

### `/grantrole`

Параметры:

- `role` - роль, которую нужно выдать;
- `include_bots` - включать ли ботов в выдачу, по умолчанию нет;
- `reason` - причина для Discord audit log, по умолчанию используется значение из `.env`.

## Что нужно в Discord Developer Portal

1. Создай application и bot.
2. Включи `SERVER MEMBERS INTENT`.
3. Пригласи бота с `bot` и `applications.commands`.
4. Дай боту право `Manage Roles`.
5. Подними роль бота выше той роли, которую он должен выдавать.

## `.env`

Файл уже создан локально. Заполни его:

```env
DISCORD_BOT_TOKEN=PASTE_YOUR_DISCORD_BOT_TOKEN_HERE
ALLOWED_GUILD_ID=PASTE_YOUR_MAIN_SERVER_ID_HERE
ROLE_GRANT_DELAY_MS=350
SKIP_BOTS_BY_DEFAULT=true
DEFAULT_GRANT_REASON=Kuruma bulk role grant after server migration
REPORTS_DIR=data/reports
```

## Установка и запуск

```bash
npm install
npm start
```

## Как использовать

1. Запусти бота.
2. На сервере введи `/grantrole`.
3. Выбери роль через Discord role picker.
4. Подтверди запуск кнопкой.
5. Проверяй прогресс через `/status`.

## Важные ограничения

- Бот не может выдать роль, которая выше его собственной роли.
- Бот не выдает managed/integration роли.
- Если участник выше роли бота, он будет пропущен как `not manageable`.
- Одновременно может идти только одна массовая задача.

## Отчеты

После каждой задачи бот пишет JSON-файл в `data/reports`. Там остаются:

- сервер и роль;
- кто запускал задачу;
- сколько участников было обработано;
- сколько успешно получили роль;
- сколько было пропущено;
- примеры ошибок, если они были.
